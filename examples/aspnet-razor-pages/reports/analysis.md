## 1. Page Model Design

**Dimension:** Thin vs fat models — do PageModels delegate to services or contain inline DbContext queries?

### no-skills — Thin (Service Delegation) ✅

```csharp
// no-skills/SparkEvents/src/SparkEvents/Pages/Events/Details.cshtml.cs
public class DetailsModel : PageModel
{
    private readonly IEventService _eventService;
    public DetailsModel(IEventService eventService) => _eventService = eventService;

    public async Task<IActionResult> OnGetAsync(int id)
    {
        Event = await _eventService.GetEventByIdAsync(id);
        if (Event == null) return NotFound();
        return Page();
    }
}
```

### dotnet-webapi — Thin (Sealed + Primary Constructor) ✅

```csharp
// dotnet-webapi/SparkEvents/src/SparkEvents/Pages/Events/Details.cshtml.cs
public sealed class DetailsModel(IEventService eventService) : PageModel
{
    public async Task<IActionResult> OnGetAsync(int id, CancellationToken ct)
    {
        Event = await eventService.GetByIdAsync(id, ct);
        if (Event is null) return NotFound();
        return Page();
    }
}
```

### dotnet-artisan — Thin (Sealed + Service Delegation) ✅

```csharp
// dotnet-artisan/SparkEvents/src/SparkEvents/Pages/Events/Details.cshtml.cs
public sealed class DetailsModel(IEventService eventService) : PageModel
{
    public async Task<IActionResult> OnGetAsync(int id, CancellationToken cancellationToken)
    {
        Event = await eventService.GetEventByIdAsync(id, cancellationToken);
        if (Event is null) return NotFound();
        return Page();
    }
}
```

### managedcode-dotnet-skills — Hybrid (DbContext + Services) ⚠️

```csharp
// managedcode-dotnet-skills/SparkEvents/src/SparkEvents/Pages/Venues/Index.cshtml.cs
public class IndexModel(SparkEventsDbContext db) : PageModel
{
    public List<Venue> Venues { get; set; } = [];

    public async Task OnGetAsync()
    {
        Venues = await db.Venues        // Direct DbContext — no service layer
            .AsNoTracking()
            .OrderBy(v => v.Name)
            .ToListAsync();
    }
}
```

managedcode injects `SparkEventsDbContext` directly into ~17 PageModels for simple CRUD (Venues, Categories, TicketTypes) while using services for complex business logic.

**Verdict:** 🥇 dotnet-webapi = dotnet-artisan (100% thin) > 🥈 no-skills (thin but traditional) > 🥉 managedcode (hybrid)

## 3. Input Model Separation

**Dimension:** Dedicated InputModel classes vs binding directly to entities

**100% adoption** across all 12 projects. Zero direct entity binding found.

```csharp
// All configs — Property Create page
[BindProperty]
public InputModel Input { get; set; } = new();

public class InputModel
{
    [Required, MaxLength(200)]
    public string Name { get; set; } = string.Empty;
    [Required, Range(1, int.MaxValue)]
    public int TotalUnits { get; set; }
}

public async Task<IActionResult> OnPostAsync()
{
    var property = new Property   // Explicit InputModel → Entity mapping
    {
        Name = Input.Name,
        TotalUnits = Input.TotalUnits
    };
    await _service.CreatePropertyAsync(property);
}
```

### dotnet-webapi Enhancement — `init` Properties

```csharp
// dotnet-webapi uses init-only properties for immutability
public sealed class InputModel
{
    [Required, MaxLength(300)]
    public string Title { get; init; } = string.Empty;  // init, not set
}
```

**Verdict:** All configs equal ✅ — dotnet-webapi earns slight edge for `init` properties

## 23. Pagination

**Dimension:** Reusable components, Bootstrap rendering, aria-label

### All Configs — Reusable PaginatedList\<T\> ✅

```csharp
// PaginatedList.cs — All configs implement this pattern
public sealed class PaginatedList<T>
{
    public IReadOnlyList<T> Items { get; }
    public int PageNumber { get; }
    public int TotalPages { get; }
    public bool HasPreviousPage => PageNumber > 1;
    public bool HasNextPage => PageNumber < TotalPages;

    public static async Task<PaginatedList<T>> CreateAsync(
        IQueryable<T> source, int pageNumber, int pageSize,
        CancellationToken ct = default)
    {
        var count = await source.CountAsync(ct);
        var items = await source
            .Skip((pageNumber - 1) * pageSize)
            .Take(pageSize)
            .ToListAsync(ct);
        return new PaginatedList<T>(items, count, pageNumber, pageSize);
    }
}
```

### Pagination Partial — Bootstrap + ARIA ✅

                </a>

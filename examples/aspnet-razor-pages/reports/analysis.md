# Comparative Analysis: dotnet-skills, no-skills, dotnet-webapi, dotnet-artisan, managedcode-dotnet-skills

## Introduction

This report compares ASP.NET Core Razor Pages applications generated by **5 different Copilot skill configurations**, each producing 3 identical scenario apps:

| Configuration | Description |
|---|---|
| **dotnet-artisan** | dotnet-artisan plugin chain (9 skills + 14 specialist agents) |
| **dotnet-skills** | Official .NET Skills (dotnet/skills) |
| **no-skills** | Baseline (default Copilot, no skills) |
| **dotnet-webapi** | dotnet-webapi skill |
| **managedcode-dotnet-skills** | Community managed-code skills |

Each configuration generated three apps:
- **SparkEvents** — Event registration portal with ticket types, capacity management, waitlists, and check-in
- **KeystoneProperties** — Property management with leases, tenants, maintenance requests, and rent tracking
- **HorizonHR** — Employee directory and HR portal with leave management, reviews, and skills

All apps target **.NET 10**, use **EF Core with SQLite**, and follow **Bootstrap 5** styling. Page counts are comparable across configurations (29–34 page models per app).

{
